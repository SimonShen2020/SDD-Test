# Risk Register - v0.2 Planning

本文件记录 v0.1.0 之后的主要风险与建议改进项，按优先级排序。

## P0（高优先）

1. 插件 ABI 稳定性风险
- 现状：插件注册接口与核心抽象仍处于早期演进阶段。
- 风险：接口调整可能导致已编译插件不兼容。
- 建议：冻结插件 C ABI 边界，新增版本协商字段。

2. Huarui 真实 SDK 行为偏差风险
- 现状：Huarui 适配层为最小骨架实现。
- 风险：真实设备流、回调线程、错误码语义可能与模拟行为不一致。
- 建议：补充 SDK 集成测试与设备在环验证。

## P1（中优先）

3. 配置解析健壮性风险
- 现状：解析器采用轻量正则实现。
- 风险：复杂 XML（注释、空白、顺序变化）下可能存在边界问题。
- 建议：引入稳定 XML 解析库并增加 schema 校验。

4. 服务层能力深度不足风险
- 现状：质量检测/嫁接/畸变矫正/四点定位为契约实现。
- 风险：业务方接入后需二次改造成本。
- 建议：定义算法输入输出模型并逐项补齐实现与性能测试。

## P2（可排期）

5. 观测性与诊断能力不足
- 现状：当前日志为简易文件日志。
- 风险：现场复杂故障定位效率有限。
- 建议：支持结构化日志、按模块级别过滤、trace id。

6. 跨平台 CI 缺失
- 现状：当前在 Windows 构建验证通过。
- 风险：Linux 编译差异可能滞后暴露。
- 建议：增加 Windows/Linux 双平台 CI 流水线。

## 建议的 v0.2 里程碑

- M1: 插件 ABI 冻结与版本协商
- M2: Huarui 真实 SDK 接入测试
- M3: XML 解析库替换 + schema 校验
- M4: 服务层算法能力逐项实装
- M5: 结构化日志与跨平台 CI
